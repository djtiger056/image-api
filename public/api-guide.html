Images-API 外部接入文档

本服务对外提供统一的 图片生成、视频生成、AI 对话 接口，兼容 OpenAI 格式。底层封装了即梦 (Jimeng)、可灵 (Kling)、豆包 (Doubao)、小云雀 (XYQ)、通义千问 (Qwen/Wan) 五大平台，支持自动轮询、额度降级。

Base URL

http://<your-server>:<port>

所有接口路径均相对于此 Base URL。部署后请替换为你的实际服务地址。端口默认 8006（可在 local.env 中通过 PORT 修改）。

Content-Type

所有 POST 请求均需设置 Content-Type: application/json，图生图文件上传使用 multipart/form-data。

1. 认证方式

本服务有两层认证：

1.1 服务级 API Key（可选）

如果服务端配置了 SERVER_API_KEY 或 SERVER_API_KEYS，所有请求需携带：

x-api-key: your-api-key
💡 说明 未配置 API Key 时此 header 可省略。控制台页面和外部调用规则一致。

1.2 上游平台凭证

用于授权上游平台（即梦、豆包等）执行生成任务。两种方式：

  • 服务端配置（推荐）：在 local.env 或 configs/accounts.json 中预置凭证，调用时无需传 Authorization
  • 请求时传入：通过 Authorization: Bearer <sessionid> 传入，多账号用逗号分隔
# 服务端已配置凭证时，无需传 Authorization
curl -X POST http://your-server:8006/v1/images/generations \
  -H "Content-Type: application/json" \
  -d '{"model":"jimeng-4.5","prompt":"一朵白色小花"}'

# 请求时传入凭证
curl -X POST http://your-server:8006/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-jimeng-sessionid" \
  -d '{"model":"jimeng-4.5","prompt":"一朵白色小花"}'

# 传入 API Key + 上游凭证
curl -X POST http://your-server:8006/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "x-api-key: your-api-key" \
  -H "Authorization: Bearer your-jimeng-sessionid" \
  -d '{"model":"jimeng-4.5","prompt":"一朵白色小花"}'

1.3 凭证获取方式

4. 图片生成（统一接口）

POST /v1/images/generations

支持文生图和图生图，兼容即梦、可灵、豆包、小云雀、千问等多个平台。通过 model 参数自动路由到对应 provider。

4.1 请求参数

model string 必填 模型名称，见下方模型列表
prompt string 必填 生成提示词，支持中文/英文
ratio string 可选 图片比例：1:1 16:9 9:16 4:3 3:4 3:2 2:3 21:9（豆包/小云雀/即梦），auto（千问），默认 1:1
resolution string 可选 分辨率：1k 2k，默认 1k。仅 wan2.7-image 支持自定义分辨率；即梦/豆包/小云雀使用平台默认分辨率（约2K），其他千问模型不支持此参数
negative_prompt string 可选 反向提示词，描述不想出现的内容
images string[] 可选 图生图：URL 数组或含 url 字段的对象数组，最多 10 张
n number 可选 生成图片数量，默认 1
sample_strength number 可选 采样强度，控制生成的随机性
intelligent_ratio boolean 可选 智能比例，让模型自动选择最佳比例
response_format string 可选 url（默认）或 b64_json
async boolean 可选 异步模式（目前仅可灵支持）
provider_options object 可选 透传给底层 provider 的额外参数
⚠️ 注意 不支持 size、width、height 参数，请统一使用 ratio 控制比例。即梦/豆包/小云雀使用平台默认分辨率（约2K），仅千问 wan2.7-image 支持通过 resolution 参数自定义 1k/2k。

5. Seedream 统一生图接口

POST /v1/seedream/generations

自动轮询 统一 Seedream 模型，底层自动在 jimeng → xyq → doubao 三个平台间轮询，积分/额度耗尽时自动降级到下一个平台。

5.1 模型列表

模型 ID说明
seedream-5.0最新版，自动轮询三平台
seedream-4.5推荐，自动轮询三平台
seedream-4.0自动轮询三平台

5.2 额外参数

style string 可选 风格参数，透传给底层 provider

其他参数与 /v1/images/generations 一致。

5.3 请求示例

curl -X POST http://your-server:8006/v1/seedream/generations \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedream-5.0",
    "prompt": "一只在月光下散步的白猫，梦幻水彩风格",
    "ratio": "16:9"
  }'
💡 推荐 不确定用哪个平台时，直接用 seedream-5.0，服务会自动选择最优可用平台。

file_paths string[] 可选 输入素材 URL 数组（图生视频/参考图）

7. 豆包视频生成

POST /v1/doubao/videos/generations

豆包 Seedance 2.0 Fast 视频生成，每日 10 次免费额度。

7.1 请求参数

model string 必填 doubao-seedance-2.0-fast
prompt string 必填 视频描述
ratio string 可选 16:9 9:16 1:1 4:3 3:4，默认 16:9。图生视频时自动推断，不要传
duration number 可选 固定 5 秒（消耗 1 额度）
images string[] 可选 参考图 URL 数组（图生视频模式）
resolution string 可选 分辨率（按上游能力）
timeout_ms number 可选 超时时间（毫秒），默认 300000
response_format string 可选 url（默认）或 b64_json

7.2 文生视频示例

curl -X POST http://your-server:8006/v1/doubao/videos/generations \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2.0-fast",
    "prompt": "一只小猫在草地上欢快地奔跑，阳光灿烂",
    "ratio": "16:9"
  }'

7.3 图生视频示例

⚠️ 重要规则：图生视频不传 ratio 当传了 images 时，不要传 ratio。服务会根据首张参考图尺寸自动推断比例。如需固定比例，请先裁剪参考图。 
curl -X POST http://your-server:8006/v1/doubao/videos/generations \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2.0-fast",
    "prompt": "保持画面风格一致，让角色缓慢转头微笑",
    "images": ["https://example.com/first-frame.jpg"]
  }'

14.2 响应格式 (response_format)

  • url（默认）— 返回图片/视频的可访问 URL
  • b64_json — 返回 Base64 编码的内容

14.3 统一结果字段

所有接口成功时至少返回：

{
  "created": 1717000000,       // Unix 时间戳
  "data": [
    { "url": "https://..." }  // 或 b64_json
  ]
}

视频接口可能额外返回：

  • videoUrl — 主视频 URL
  • videoUrls — 所有视频 URL 数组
  • result_url — 备用 URL 字段
  • model — 实际使用的模型
  • provider — 实际使用的平台
💡 兼容建议 业务侧请优先读取 data[0].url，并兼容 videoUrl、result_url、videoUrls[0]。

15. 错误处理

15.1 错误响应格式

{
  "error": {
    "message": "错误描述信息",
    "type": "错误类型"
  }
}

15.2 常见错误类型

type说明处理方式
invalid_request_error请求参数错误检查 model、prompt 等必填参数
authentication_error认证失败检查 x-api-key 或 Authorization
quota_exhausted额度用完换账号或等次日重置
generation_failed生成失败重试或更换模型
timeout生成超时增加 timeout_ms 或重试

15.3 HTTP 状态码

  • 200 — 成功
  • 400 — 请求参数错误
  • 401 — 认证失败（API Key 无效）
  • 403 — 无权限
  • 500 — 服务端内部错误

16. 接入建议

16.1 推荐架构

Browser → 你的后端服务 → images-api → 上游平台
🔒 安全提醒 不要将凭证（sessionid、API Key）暴露给普通浏览器用户。推荐通过你的后端服务中转调用 images-api。

16.2 图片生成推荐

  • 不关心具体平台：用 seedream-5.0（/v1/seedream/generations），自动轮询三平台
  • 免费生图：用 qwen-image-2.0，千问免费额度
  • 指定平台：用对应模型如 jimeng-4.5、doubao-seedream-4.5

16.3 视频生成推荐

  • 免费体验：doubao-seedance-2.0-fast（每日 10 次）
  • 正式文生视频：wan2.7-t2v（免费、高质量、支持 1080P）
  • 首帧参考图视频：wan2.7-t2v 或 doubao-seedance-2.0-fast
  • 长时任务/异步：使用 /v1/videos/generations/async

16.4 图生视频重要规则

⚠️ 图生视频不传 ratio 当请求里传了 images（参考图），不要传 ratio。服务会读取首张参考图尺寸并自动推断最接近的视频比例。如需固定比例，请先将参考图裁剪到目标比例再传。

16.5 完整接入流程

  1. 调用 GET /v1/models 确认服务可用、获取模型列表
  2. 选择合适的模型和接口
  3. 调用生成接口，获取结果 URL
  4. （视频接口）如需异步，先调用 /v1/videos/generations/async 提交，再轮询 GET /v1/videos/generations/async/:taskId
  5. 业务侧优先读取 data[0].url，兼容其他 URL 字段

16.6 TypeScript / JavaScript 快速对接

const API_BASE = "http://your-server:8006";

async function generateImage(prompt: string, model = "seedream-5.0") {
  const res = await fetch(`${API_BASE}/v1/images/generations`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      // 如需 API Key: "x-api-key": "your-key",
    },
    body: JSON.stringify({ model, prompt, ratio: "16:9", resolution: "2k" }),
  });
  const data = await res.json();
  if (data.error) throw new Error(data.error.message);
  return data.data[0].url; // 图片 URL
}

async function generateVideo(prompt: string, model = "wan2.7-t2v") {
  const res = await fetch(`${API_BASE}/v1/qwen/videos/generations`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model, prompt,
      ratio: "16:9",
      resolution: "1080P",
      duration: 5,
    }),
  });
  const data = await res.json();
  if (data.error) throw new Error(data.error.message);
  return data.data[0].url; // 视频 URL
}

import requests

API_BASE = "http://your-server:8006"

def generate_image(prompt, model="seedream-5.0"):
    resp = requests.post(f"{API_BASE}/v1/images/generations", json={
        "model": model,
