# kling-api 启动与关闭文档

项目根目录：`/myproject/kling-api`
管理说明：
 以后管理用这些命令：

     sudo systemctl status images-api    # 查看状态
     sudo systemctl restart images-api   # 重启
     sudo systemctl stop images-api      # 停止
     sudo journalctl -u images-api -f    # 看日志

     改代码后只需 build + restart：

     cd /myproject/images-api && npm run build && sudo systemctl restart images-api
     

     然后用这个命令看带颜色的日志：

     sudo journalctl -u images-api -f --output=cat

## 1. 安装依赖

```bash
cd /myproject/kling-api
npm install
```

如果需要使用 Kling 网页模式，建议提前安装 Chromium：

```bash
npx playwright-core install chromium --with-deps
```

## 2. 开发模式启动

```bash
cd /myproject/kling-api
npm run dev
```

说明：
- 会开启 tsup 监听编译
- 编译成功后自动启动服务
- 默认端口是 `8000`

## 3. 生产模式启动

先构建：

```bash
cd /myproject/kling-api
npm run build
```

再启动：

```bash
cd /myproject/kling-api
npm start
```

## 4. 关闭服务

### 如果是前台运行
直接按 `Ctrl + C`。

### 如果是后台运行
先查进程，再结束：

```bash
ps -ef | grep "dist/index.js" | grep -v grep
kill <PID>
```

如果进程没退，再强制结束：

```bash
kill -9 <PID>
```

### 如果你是用 Docker 跑的
查看容器：

```bash
docker ps --filter name=jimeng-free-api-all
```

停止容器：

```bash
docker stop jimeng-free-api-all
```

删除容器：

```bash
docker rm jimeng-free-api-all
```

## 5. 常用检查

健康检查：

```bash
curl http://127.0.0.1:8000/ping
```

查看是否占端口：

```bash
ss -lntp | grep 8000
```

## 6. 常见环境变量

Kling 网页模式常用：

```bash
export KLING_WEB_HEADLESS=true
export KLING_WEB_TARGET_URL="https://kling.ai/app/image/new"
export KLING_WEB_ARTIFACTS_DIR="/lf/kling-web-artifacts"
export TMPDIR="/lf/tmp-downloads"
```

如果你的浏览器可执行文件路径不是默认值，还可以设置：

```bash
export BROWSER_EXECUTABLE_PATH="/usr/bin/google-chrome-stable"
```

## 7. 推荐启动顺序

```bash
cd /myproject/images-api
npm run build
npm start
```

如果要长期后台运行，建议配合 `tmux`、`screen` 或你自己的进程守护方式。
